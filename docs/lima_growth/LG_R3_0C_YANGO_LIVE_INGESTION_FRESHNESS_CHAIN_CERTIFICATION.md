## 6. GAP EXPLANATION

### Why snapshot=06-05 but raw_orders=06-01?

```
Yango API raw orders:     max 2026-06-01 (STALE - no fresh ingestion)
driver_history_weekly:    max 2026-06-01 (bootstrapped from trips_2025/2026)
driver_state_snapshot:    max 2026-06-05 (built from history_weekly + pipeline)

EXPLANATION:
  The snapshot is generated by build_driver_state_snapshot() which reads
  from driver_history_weekly (PRIMARY source). The history was bootstrapped
  from public.trips_2025/2026 tables which contain data through 2026-06-01.
  
  The pipeline was manually executed for 2026-06-03, 06-04, and 06-05
  (R1.4 recovery). The snapshot for these dates reused the same history data
  (since no new trips had occurred). This is why the snapshot shows 06-05
  while raw orders only go to 06-01.
  
  This is NOT a data fabrication. The snapshot correctly shows the driver
  state AS OF that date using the available historical data.
```
